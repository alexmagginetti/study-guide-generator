--- How these were produced, and how anyone can verify them ---

These are plain SHA-256 hashes of the exact file bytes as stored in the
canonical Git repository and served by GitHub (LF line endings) — the same
bytes the bundle-build GitHub Action and any web download receive. This
basis was chosen deliberately: it is reproducible on any operating system
by anyone, which is what makes the verification trustworthy. (A Windows
working copy checked out with core.autocrlf=true will differ by line
endings only; that is an end-of-line artifact, not tampering. Verify a
file as downloaded from GitHub, not a CRLF-converted local checkout.)

Reproduce any line yourself (example shown for GENERATOR.md):

  # Any OS with git + sha256sum, against the live canonical repo:
  git show main:GENERATOR.md | sha256sum

  # Or against a file downloaded from GitHub "raw":
  curl -sL https://raw.githubusercontent.com/alexmagginetti/study-guide-generator/main/GENERATOR.md | sha256sum

  # Windows PowerShell, against a file you downloaded:
  Get-FileHash .\GENERATOR.md -Algorithm SHA256

Compare the result to the matching line above (comparison is
case-insensitive). If a file you received does not match, do not use it —
re-download from https://github.com/alexmagginetti/study-guide-generator
and, if it still differs, report it via a GitHub Issue.

Notes: CHECKSUMS.txt does not hash itself. BACKLOG.md is intentionally
excluded — it is a maintainer process log, not a file the kit's AI
consumes. STUDY_KIT_BUNDLE.md is a derived artifact regenerated by the
build-bundle GitHub Action on merge to main; its hash above reflects the
bundle as of this kit version.
